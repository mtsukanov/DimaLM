# Спецификация: Алгоритм динамического управления аватарами

**Цель:** Обеспечение визуального соответствия состояния интерфейса (аватара) текущей фазе взаимодействия (активный диалог или простой).

## 1. Логика режима "Простой" (Idle State)
Данный режим активируется при обработке реплик под индексами **№1** и **№8** из файла `@reply`, либо при отсутствии входящего потока сообщений в течение заданного интервала. Выбор аватара зависит от переменной $\Delta T$ (время, прошедшее с момента последнего зафиксированного события/реплики).

| Условие ($\Delta T$) | Состояние системы | Аватар | Эмоция |
| :--- | :--- | :--- | :--- |
| **$\Delta T < 5$ секунд** | Кратковременная пауза (активное ожидание) | `avatar_1.jpeg` | Нейтральный |
| **$\Delta T \ge 5$ секунд** | Длительный простой (режим ожидания) | `avatar_8.jpeg` | Усталый |

## 2. Логика режима "Активный диалог" (Active State)
Применяется ко всем входящим репликам, кроме индексов №1 и №8 (т.е. для всех $i \in \{2\dots7, 9\dots18\}$). Выбор аватара производится на основе семантического анализа тональности (sentiment analysis) текста сообщения.

| Семантический кластер (ключевые маркеры) | Аватар | Эмоция |
| :---            | :--- | :--- |
| **Подтверждение/Внимание** (*"ага", "да", "понял", "в планах"*) | `avatar_3.jpeg` | Внимательный |
| **Скука/Игнорирование** (*"не успеваю", "не до этого"*) | `avatar_4.jpeg` | Скучающий |
| **Неопределенность/Ошибка** (*"опс", "не помню", "сложно оценить"*) | `avatar_5.jpeg` | Растерянный |
| **Фокус/Деловая активность** (*"спасиб", "обсуждать", "решение принято"*) | `avatar_6.jpeg` | Сосредоточенный |
| **Дружелюбие/Эмоции** (*"блин", "игриво", смайлы ")"*) | `avatar_7.jpeg` | Игривый |
| **Агрессия/Негатив** (*"агрессивно", "гемор"*) | `avatar_9.jpeg` | Гневный |
| **Задумчивость (Fallback)** *(если тональность не определенa)* | `avatar_2.jpeg` | Задумчивый |

## 3. Технические требования к реализации
1.  **Мониторинг времени:** Система должна поддерживать непрерывное отслеживание метки времени (`timestamp`) последнего полученного сообщения для вычисления $\Delta T$.
2.  **Приоритет условий:** Проверка временного порога ($\Delta T$) для реплик №1 и №8 имеет наивысший приоритет перед анализом семантики.
3.  **Обработка исключений:** В случае невозможности парсинга текста или ошибки семантического анализа, система должна использовать `avatar_2.jpeg` (Задумчивый) как безопасное дефолтное состояние.
