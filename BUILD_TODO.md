# План сборки приложения "Dima Chat"

## 1. Подготовка данных (Pipeline)
- [ ] **Парсинг `replies_mapping.md`**: Извлечь строки из таблицы Markdown и преобразовать их в массив JavaScript объектов `DI_REPLIES` (id, text, avatar, emotion).
- [ ] **Кодирование ассетов (Base64)**: Считать все изображения из директории `avatars/`, закодировать их в Base64 и сохранить в константу `ASSETS`.

## 2. Frontend: Визуальный движок (HTML/CSS)
- [ ] **Эстетика "Cyber-Nostalgia"**:
    - [ ] Настроить типографику (моноширинный шрифт с характерным межбуквенным интервалом).
    - [ ] Реализовать эффект CRT-монитора (сканирующие линии, легкий шум/зернистость через CSS).
    - [ ] Создать цветовую схему (глубокий угольный фон + фосфорно-зеленый или янтарный акцент).
- [ ] **Структура интерфейса**:
    - [ ] Контейнер чата (max-width: 600px, центрирование).
    - [ ] Область вывода сообщений с поддержкой автоматической прокрутки вниз.
    - [ ] Поле ввода и кнопка отправки (стиль терминала).

## 3. JavaScript: Логика управления (The Brain)
- [ ] **Движок машинки (Typewriter Engine)**: Реализовать вывод текста со скоростью 5 символов в секунду ($200\text{мс}$ на символ).
- [ ] **Автомат состояний аватара (ASM)**:
    - [ ] Логика переключения `img.src`.
    - [ ] Реализация CSS-переходов (`transition`) и эффекта "глитча" при смене эмоций.
- [ ] **Мониторинг $\Delta T$**:
    - [ ] Создать таймер (heartbeat) каждые $100\text{мс}$.
    - [ ] Реализовать условие: если бездействие $\ge 5.0$ сек $\to$ переход в режим "Устал" (`avatar_8.jpeg`).
- [ ] **Обработка сообщений**:
    - [ ] Функция `handleUserMessage` (сброс таймера $\Delta T$, очистка буфера при быстром вводе).
    - [ ] Рандомизация ответов на основе массива `DI_REPLIES`.
- [ ] **Локализация**: Проверить, что все элементы UI и плейслондеры на русском языке.

## 4. Финализация и проверка (Validation)
- [ ] **Автономность**: Убедиться, что `index.html` не делает никаких внешних сетевых запросов.
- [ ] **Сборка**: Объединить всё в один файл `index.html`.
- [ ] **Тестирование**: Проверить соблюдение всех технических ограничений (скорость печати, порог 5 секунд, отсутствие ошибок в консоли).
